editfriendgroups

Name

editfriendgroups — Edit the user's defined groups of friends.

Mode Description

Given several optional lists, will add/delete/update/rename the friends groups for a user.

Arguments

  1. [struct](required) Containing keys:

    • username:

      • [scalar](required) Username of user logging in.

    • password:

      • [scalar](required) Password of user logging in in plaintext. Either this or "hpassword" must be present.

    • hpassword:

      • [scalar](required) MD5 digest of user's password. Not much more secure than password, but at least it's not in plain text.

    • ver:

      • [scalar](required) Protocol version supported by the client; assumed to be 0 if not specified. See Chapter 8 for details on the protocol version

    • groupmasks:

      • [struct](required) Containing keys:

      • set:

        • [struct](required) Containing keys:

          • [struct](optional; multiple allowed) Containing keys:

            • name:

              • [scalar](required) Create or rename the friend group by sending this key. The value is the name of the group.

            • sort:

              • [scalar](required) This field should be sent to indicate the sorting order of this group. The value must be in the range of 0-255. The default is 50.

            • public:

              • [scalar](required) If this is "1", then this group is marked as public. If public, other users can see the name of the group and the people that are in it.

      • delete:

        • [list](required) Containing items:

          • [scalar](optional; multiple allowed) A number of a friend group to delete (which can be from 1-30, inclusive). The server will modify all old entries that allow access to that friend group, so a new friend group using that number won't have access to old non-related entries. However, it is the client's responsibility to modify each friend's groupmask to remove them from the group as it's deleted.
